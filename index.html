User Guide


Tab Overview

  • Owner: view and modify owners of the file/folder
  • Change Permissions: modify permissions for users/groups using “allow” and “deny.” You can also:
    • Include inheritable permissions from this object's parent
    • Replace all child object permissions with inheritable permissions from this object
  • View Permissions: view the inherited permissions on files/folders for users/groups.

Preparation

  1. Read the prompt and determine:
    • Which file or folder do you need to modify permissions for?
    • Which user or group?
    • Which permissions? Is it a selected few (read, write, etc.) or all the permissions?
  2. Find the appropriate file/folder and click on the Permissions lock button
  3. Reveiew “Owners” and “View Inheritance” and consider what you see for the next step
  4. Modify permissions for the user/group in Change Permissions (see specific instructions for each type of modification below)
    • Add permissions
    • Remove permission
    • Fix permission

Add Permission

  1. Does the user already have access to the file/folder?
    • If so, proceed to next step
    • If not, go to “Change Permissions” to add the user/group or go to “Owner” to make the user/group the owner
  2. Go to “Change Permissions”, click “allow” to add the direct permissions specified in the prompt AND consider the following:
    • Are there inherited permissions (light gray checkboxes)?
      • If not, consider adding “include inheritable permissions from this object's parent” OR “replace all child object permissions with inheritable permissions from this object”
    • Is the user part of a greater group (e.g. employees)?
      • If so, modify permissions for the group in “Change Permissions”
      • If not (individual user), modify permisisons for the individual

Remove Permission

  1. Do you need to remove all (not some) permissions?
    • If so, remove the user/group entirely in “Change Permissions” or change the owner in “Owner”
  2. Go to “Change Permissions”, click “deny” to remove the direct permissions specified in the prompt AND consider the following:
    • Are there inherited permissions (light gray checkboxes)?
      • If there are and “include inheritable permissions from this object's parent” OR “replace all child object permissions with inheritable permissions from this object” is checked off, consider unchecking the box to modify specific permissions for the user
    • Is the user part of a greater group (e.g. employees)?
      • If so, remove permissions for the group in “Change Permissions”
      • If not (individual user), remove permisisons for the individual

Fix Permission

  1. Determine where the problem lies — do you need to add or remove permissions?
    • Clues: are they able to modify something they are not supposed to? Or are they not able to modify something they should be able to?
  2. Go to either Add Permissions or Remove Permissions
    • Pay special attention to the “consider the following” options
